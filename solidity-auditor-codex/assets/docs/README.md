# Project Docs

Use this directory for local context that helps explain how the protocol is supposed to behave.

Good inputs include:

- design docs and specs
- intended invariants
- plain-English descriptions of protocol behavior
- known limitations or accepted tradeoffs

## Supported Inputs

- Plain text files
- Markdown files
- Short notes focused on the module currently under review

## Manual Workflow

This migrated skill does not auto-fetch online documentation.

- If you place URL lists here, treat them as manual reading aids only.
- If online docs matter for the review, fetch them yourself and save the relevant excerpts locally before running the audit.
- Prefer concise local notes over large background dumps.
