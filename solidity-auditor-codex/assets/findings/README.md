# Findings

Use this directory as the report archive for `solidity-auditor-codex`.

It may contain:

- markdown reports generated by `bash scripts/write-audit-report.sh`
- historical reports from earlier runs
- external audit reports or manual review notes

## Report Path Contract

Generated reports should use:

`assets/findings/{project-name}-pashov-ai-audit-report-{timestamp}.md`

where `{timestamp}` is `YYYYMMDD-HHMMSS`.

## Manual Re-Verification Only

The migrated skill does not auto-read every file in this directory and does not auto-carry findings forward.

If you want to compare against older findings:

1. Read the relevant prior report manually.
2. Re-check each issue against the current codebase.
3. Carry it forward only when the exploit path still exists.
4. Note the historical status explicitly in the new report if that context matters.
